What is testing?

testing is the process of making sure our program is correct. its used in many forms all over the industry but here are the basics

What are the types of testing?

Iterative

happens while the program is being developed usually testing one bit of functionality

Final or black box

This happens near the end of a programs development. this is to test the program in its whole to catch any bugs iterative testing missed, its also known as black box testing as we are not worried about what the program is doing inside the box just that it does work

Types of test data

Normal

this is the normal data expected, if this fails you have a big problems

Boundary

this is data on the edge of what is acceptable, lets say you need to take in a date to show that people are over the age of 18. you would need to discard dates after 3rd march 2004 but that date is also valid and needs to be checked.

invalid

this is data that is of the same type (so in this example a date) but is otherwise wrong

erroneous

this is data that is of the wrong type and should be discarded always

Refining algorithms

this is usually in realtion to fining things that are wrong and correcting them but can also relate to makeing algo’s more efficent
