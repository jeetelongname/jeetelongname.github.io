Loops

Its all well and good to have a collection of values, but what if we want to use this collection? maybe print it out?. or more generally, how do we write code that will be run many times?

This is where loops come in. Loops are a statement that allows us to run code many times, either a definite amount of times (for ten iterations as an example) or while a value remains true.

For loops

There are 2 kinds of for loop you need to know about. The count controled loop and the collection loop

Count Controlled

Lets say we want to iterate 10 times we can use the range construct. This gives us a range we can move over. 

for i in range(10):
    print(i)

Adding in some more details

We can also add in an optional start value and a step, like so 

for i in range(start := 2, stop := 12, step := 2):
    print(i)

While loops

While loops are simpler, they just need a boolean expression These loops can run for an indefinite amount of time, because the boolean expression may never become false 

a = 0
while a != 10:
    print(a)
    a += 1

a = 0
while a != 10:
    print(a)

Classes

We have disucssed a lot of things, If you cast your mind back we discussed types, Those types have operations to them whether its add for numbers or the .upper() operation for strings

Let us discuss a thought experiment. Lets say we want to model a bin. The Bin has a few features. It tells us how full it is. what it can store.

What do we need to model this bin?

Well we need to be able to create a bin.

make a bin add trash to the bin empty the bin

so lets make a make_bin function! 

def make_bin(type_of_bin):
    return [0, type_of_bin]

def add_rubbish(bin, rubish_type):
    if bin[1] == rubish_type:
        bin[0] += 1
    else:
        print("Use a different bin!")
    return bin

def empty_bin(bin):
    bin[0] = 0
    return bin

now lets use this! 

bin = make_bin("glass")
bin = add_rubbish(bin, "glass")
bin = add_rubbish(bin, "plastic")
bin = empty_bin(bin)

Whats the problem with this?

We have to manually pass in the bin and capture it again. None of these functions are connected. If we pass in a thing that looks like a bin it will still work which is a problem because it may break our system.

What we want is a new type that encapsulates this nice and cleanly

  • this may be a problem

A class for our bin

to create a new type we need to use the class keyword. 

class Name:

class Bin:

class Bin:
   def __init__(self, bin_type):
       self.ammount = 0
       self.bin_type = bin_type

   def add_rubbish(self, rubbish_type):
       if self.bin_type == rubbish_type:
          self.ammount += 1
       else:
          print("Use a different bin!")

   def empty_bin(self):
      self.ammount = 0
